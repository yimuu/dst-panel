# Frontend Build Artifacts Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep frontend build artifacts under `web-ui/dist`, remove committed root `dist` assets from Git, and keep Docker/runtime static serving working from packaged `/app/dist`.

**Architecture:** Vite builds into `web-ui/dist`. Docker images build the frontend from `web-ui` during image build and copy `web-ui/dist` into `/app/dist`; existing entrypoints still refresh `/data/dist` from `/app/dist` at startup. Release tests validate the build pipeline instead of requiring committed root `dist` files.

**Tech Stack:** React, Vite, npm, Dockerfile, Rust release tests.

## Global Constraints

- Do not commit generated frontend artifacts from root `dist` or `web-ui/dist`.
- Preserve backend runtime behavior that serves frontend files from `root_path/dist`.
- Preserve Docker runtime behavior that copies `/app/dist` to `/data/dist`.
- Use TDD for release behavior changes: update tests first, verify they fail, then implement.

### Task 3: Build Frontend in Docker Images

**Files:**
- Modify: `Dockerfile`
- Modify: `scripts/docker/Dockerfile`
- Modify: `script/docker-build-mac/Dockerfile`
- Modify: `scripts/docker-build-mac/Dockerfile`

**Interfaces:**
- Consumes: `web-ui/package.json`, `web-ui/package-lock.json`, and the `web-ui` source tree.
- Produces: `/app/dist` inside the runtime image.

- [ ] **Step 1: Add a frontend build stage**

At the top of each Dockerfile, add:

```dockerfile
FROM node:24-bookworm-slim AS frontend-build
WORKDIR /app/web-ui
COPY web-ui/package.json web-ui/package-lock.json /app/web-ui/
RUN npm ci
COPY web-ui /app/web-ui
RUN npm run build
```

- [ ] **Step 2: Copy built assets from the stage**

Replace:

```dockerfile
COPY dist /app/dist
```

with:

```dockerfile
COPY --from=frontend-build /app/web-ui/dist /app/dist
```

- [ ] **Step 3: Keep entrypoint behavior unchanged**

Do not change `cp -a "$APP_DIR/dist/." "$DATA_DIR/dist/"`; runtime still expects `/app/dist`.
